# 요구사항 명세서

---

## 1. 서비스 개요

본 애플리케이션은 공공 해양 예보 API와 사용자 리뷰 데이터를 활용하여 바다여행, 해수욕, 바다낚시, 갯벌체험, 스킨스쿠버, 서핑에 적합한 해양 스팟을 검색·비교·추천하는 서비스입니다. 사용자는 체험 종류와 날짜를 기준으로 해양 스팟을 탐색하고, 상세 정보를 확인하며, 리뷰와 찜하기 기능을 통해 개인화된 정보를 관리할 수 있습니다.

## 2. 공통 정책 및 용어

- **스팟 / OceanSpot:** 사용자가 조회할 수 있는 해양 활동 장소를 의미하며, 데이터베이스에서는 `OceanSpot` 엔티티로 관리합니다.
- **OceanSpot 매핑 기준:** 공공 API에서 제공되는 장소명, 체험 유형, 위도(`lat`), 경도(`lot`) 또는 원본 식별값을 기준으로 `OceanSpot` 고유 ID를 생성하거나 매핑합니다. 리뷰와 찜하기 데이터는 이 `OceanSpot` ID에 연결됩니다.
- **좌표 필드 정책:** 공공 API 원본 필드가 `lat`, `lot`인 경우 그대로 수신하되, 프론트엔드와 내부 로직에서는 `latitude`, `longitude` 형태로 표준화하여 사용할 수 있습니다.
- **로그인 권한 정책:** 비로그인 사용자는 장소 검색, 지도 조회, 상세 정보 조회가 가능하며, 리뷰 작성·수정·삭제, 찜하기, 마이페이지 접근, 회원 정보 수정은 로그인한 사용자만 이용할 수 있습니다.
- **리뷰 권한 정책:** 리뷰 수정 및 삭제는 작성자 본인만 가능하며, 필요 시 관리자는 부적절한 리뷰를 삭제할 수 있습니다.
- **위치 권한 예외 처리:** 사용자가 위치 정보 제공을 거부한 경우, 현재 지도 중심 좌표 또는 서비스 기본 기준 좌표를 사용하여 가까운 순 정렬을 수행합니다.
- **공공 API 예외 처리:** 공공 API 호출 실패, 응답 지연, 해당 날짜 데이터 없음 등의 상황에서는 사용자에게 안내 메시지를 표시하고, 가능한 경우 최근 조회 가능 날짜 데이터 또는 DB 캐시 데이터를 대체 표시합니다.

## 3. 기능 요구사항

### 3.1. 회원 관리

- **회원가입:** 사용자의 고유 ID, 암호화된 비밀번호, 닉네임, 이메일을 입력받아 유효성 검사 후 `User` 테이블에 저장합니다.
- **로그인:** 입력된 ID와 비밀번호를 검증하여 세션 또는 JWT 토큰을 발급하고 로그인 상태를 프론트엔드(Vue.js)에 전달합니다.
- **로그아웃:** 로그인한 사용자가 로그아웃하면 발급된 인증 정보를 만료시키고 비로그인 상태로 전환합니다.

### 3.2. 메인 화면

- **조건별 검색 기능:**
    - **체험 종류 선택:** 셀렉트 박스나 라디오 버튼을 통해 [바다여행, 해수욕, 바다낚시, 갯벌체험, 스킨스쿠버, 서핑] 중 하나를 선택합니다. 기본값은 바다여행이며, 선택한 테마에 따라 호출할 백엔드 API 엔드포인트가 달라집니다.
    - **체험 날짜 선택:** 캘린더 컴포넌트를 통해 날짜를 선택합니다. 선택한 날짜는 공공 API의 `predcYmd` 필드와 매핑되어 예보 데이터를 필터링하는 데 사용됩니다.
    - **지도에서 해변 선택:** Kakao 또는 Naver Map API를 연동합니다. 메인 화면 진입 시 또는 체험 종류 변경 시, API 응답의 위도(`lat`)와 경도(`lot`) 데이터를 기반으로 해당 체험이 가능한 전국 해안 위치를 지도에 마커로 렌더링합니다. 마커를 클릭하면 당일 핵심 지수, 등급, 상세 정보 보기 버튼을 표시합니다.
    - **장소 미선택 시 진입 로직:** 상단 바 또는 TOP 목록 하단의 모두 보기 버튼을 클릭하면 선택한 체험 종류와 날짜 조건을 유지한 채 종합 탐색 및 비교 페이지로 이동합니다.
- **체험하기 좋은 장소 TOP 6:**
    - 사용자가 선택한 체험 종류의 API 응답 데이터 중 종합 지수(`totalIndex`)가 높은 순으로 정렬하여 상위 6개 장소를 카드 UI로 제공합니다.
    - 지수 정렬 로직은 매우좋음(5) > 좋음(4) > 보통(3) > 나쁨(2) > 매우나쁨(1) 순으로 백엔드에서 가중치 점수화하여 처리합니다.
    - 지수 점수가 동일한 경우, 사용자의 현재 위치 또는 지도 중심 좌표와 공공데이터의 위·경도 간 직선거리를 계산하여 가까운 순으로 우선 추천합니다.
- **핫한 장소 TOP 6:**
    - 자체 데이터베이스의 `Review` 테이블을 기반으로 작동합니다.
    - 현재 시스템 날짜 기준 최근 30일 이내에 생성된 리뷰 수가 가장 많은 `OceanSpot` 고유 ID 상위 6개를 그룹화 및 정렬하여 메인 화면에 배치합니다.

### 3.3. 종합 탐색 및 비교 페이지

- 메인에서 장소를 정하지 못한 사용자가 전국 해양 스팟을 조건별로 정렬하고 비교할 수 있는 허브 페이지입니다.
- **좌측: 필터 및 리스트 영역**
    - **사이드 필터:** 메인에서 전달받은 체험 종류와 날짜(`predcYmd`)가 기본 세팅되며, 페이지 내에서 다른 체험 종류와 날짜로 자유롭게 변경할 수 있습니다.
    - **전국 스팟 리스트 정렬:** 선택한 체험 API의 `totalIndex`를 백엔드에서 점수화하여 지수 좋은 순, 가까운 순, 리뷰 많은 순으로 정렬합니다. 사용자는 콤보박스를 통해 정렬 방식을 선택할 수 있습니다.
    - **스팟 카드 UI 데이터 매핑:** 리스트의 각 카드에는 API에서 제공하는 핵심 요약 정보를 표시합니다.
        - 서핑 예시: 장소명(`surfPlcNm`), 파고(`avgWvhgt` m), 등급(`grdCn`), 서핑지수(`totalIndex`)
        - 바다낚시 예시: 위치(`seafsPstnNm`), 추천 대상어(`seafsTgfshNm`), 물때(`tdlvHrCn`), 바다낚시지수(`totalIndex`)
- **우측: 와이드 지도 영역**
    - **동적 마커 바인딩:** 필터링된 해당 테마의 전국 스팟을 위도와 경도 기반으로 지도 위에 표시합니다.
    - **지수별 마커 색상 차별화:** `totalIndex`가 좋음 또는 매우좋음인 곳은 파란색, 보통은 녹색, 나쁨 또는 매우나쁨은 빨간색 마커로 표시합니다.
    - **스팟 비교 팝업:** 지도 마커를 클릭하면 커스텀 오버레이를 표시하고, 해당 장소의 당일 핵심 지수, 등급, 상세 정보 보기 버튼을 제공합니다.

### 3.4. 상세 페이지

사용자가 메인 화면의 지도 마커, TOP 6 카드, 종합 탐색 페이지의 카드 또는 지도 마커를 클릭했을 때 진입합니다. 선택된 장소와 매핑된 공공데이터의 핵심 정보를 아이콘, 게이지 바, 카드 UI 등으로 직관적으로 제공합니다.

- **테마별 맞춤 데이터 바인딩:**
    1. **바다여행 선택 시:** 위치(`sareaDtlNm`), 물때(`tdlvHrCn`), 기온(`avgArtmp`), 풍속(`avgWspd`), 수온(`avgWtem`), 파고(`avgWvhgt`), 유속(`avgCrsp`), 날씨(`weather`) 정보를 제공합니다.
    2. **해수욕 선택 시:** 해수욕장명(`bbchNm`), 개장 여부(`opnStat`), 최고파고(`maxWvhgt`), 수온(`avgWtem`), 풍속(`maxWspd`), 해수욕지수(`totalIndex`)를 제공합니다.
    3. **바다낚시 선택 시:** 위치(`seafsPstnNm`), 대상어종(`seafsTgfshNm`), 물때(`tdlvHrCn`), 파고 범위(`minWvhgt`~`maxWvhgt`), 수온 범위, 풍속 범위, 바다낚시지수(`totalIndex`)를 제공합니다.
    4. **갯벌체험 선택 시:** 갯벌체험장명(`mdftExpcnVlgNm`), 날씨(`weather`), 체험 가능 시간대(`mdftExprnBgngTm`~`mdftExprnEndTm`), 갯벌체험지수(`totalIndex`)를 제공합니다.
    5. **스킨스쿠버 선택 시:** 체험장명(`skscExpcnRgnNm`), 물때(`tdlvHrCn`), 파고·유속·수온 범위, 스킨스쿠버 지수(`totalIndex`)를 제공합니다.
    6. **서핑 선택 시:** 서핑장소명(`surfPlcNm`), 파고(`avgWvhgt`), 파주기(`avgWvpd`), 풍속(`avgWspd`), 추천 등급(`grdCn`), 서핑지수(`totalIndex`)를 제공합니다.
- **장소 리뷰 연동:**
    - 상세 페이지 하단에 해당 `OceanSpot`에 연결된 평점, 리뷰 텍스트, 작성자 닉네임, 작성일을 조회하여 리스트로 표시합니다.
    - 로그인한 사용자는 비동기 요청(Axios)을 통해 리뷰를 작성할 수 있습니다.
    - 본인이 작성한 리뷰는 상세 페이지 또는 마이페이지에서 수정·삭제할 수 있습니다.
- **찜하기 기능:**
    - 로그인한 사용자는 상세 페이지에서 해당 장소를 찜하거나 찜 해제할 수 있습니다.
    - 찜한 장소는 마이페이지의 찜한 장소 목록에 반영됩니다.

### 3.5. 마이페이지

- **나의 대시보드 및 등급 배지:**
    - 사용자가 작성한 총 리뷰 수, 찜한 장소 수 등을 DB에서 집계하여 개인 프로필 영역에 노출합니다.
    - 사용자의 리뷰 데이터를 체험 유형별로 분석하여 가장 비율이 높은 활동에 맞는 해양 배지 또는 칭호를 제공합니다.
    - 예를 들어 서핑 리뷰 비율이 가장 높고 해당 리뷰가 일정 개수 이상이면 '프로 서퍼', 낚시 리뷰 비율이 가장 높으면 '도시 어부' 배지를 부여합니다.
- **찜한 장소 및 실시간 정보 매핑:**
    - 사용자가 즐겨찾기한 해양 스팟 목록을 보여줍니다.
    - 단순히 장소명만 나열하지 않고, 해당 스팟의 실시간 또는 최신 종합 지수(`totalIndex`)를 외부 API와 매핑하여 카드 UI로 시각화합니다.
    - 외부 API 데이터가 없을 경우 최근 조회된 캐시 데이터 또는 데이터 없음 상태를 표시합니다.
- **내가 쓴 리뷰 저장 및 관리:**
    - 사용자가 상세 페이지에서 작성한 리뷰 목록을 장소명, 평점, 텍스트 내용, 작성일과 함께 조회합니다.
    - 이 페이지에서 본인이 작성한 리뷰를 즉시 수정하거나 삭제할 수 있습니다.
- **회원 정보 수정 및 탈퇴:**
    - 비밀번호 변경 및 닉네임 변경 기능을 제공합니다.
    - 회원 탈퇴 시 찜하기 정보와 작성 리뷰 데이터는 프로젝트 정책에 따라 삭제합니다. 본 프로젝트에서는 데이터베이스 무결성을 위해 `ON DELETE CASCADE` 구조를 적용하여 해당 사용자가 남긴 찜하기 정보와 리뷰 데이터를 연쇄 삭제 처리합니다.

## 4. 비기능 및 예외 요구사항

- **응답 성능:** 메인 화면과 종합 탐색 페이지는 공공 API 호출 및 DB 조회 결과를 조합하므로, 필요한 경우 백엔드 캐싱을 적용하여 응답 속도를 개선합니다.
- **데이터 정합성:** 공공 API 데이터와 내부 DB의 `OceanSpot` 매핑이 실패한 경우 임시 조회용 데이터로 표시하되, 리뷰와 찜하기 기능은 고유 ID가 확정된 장소에 대해서만 제공합니다.
- **입력 검증:** 회원가입, 로그인, 리뷰 작성, 회원 정보 수정 요청은 백엔드에서 필수값, 길이, 형식, 권한을 검증합니다.
- **사용자 안내:** 위치 권한 거부, API 데이터 없음, 네트워크 오류, 로그인 필요 상황은 사용자에게 명확한 안내 메시지로 표시합니다.
